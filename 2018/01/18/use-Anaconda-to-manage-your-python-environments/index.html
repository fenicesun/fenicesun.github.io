使用Anaconda来管理python环境

Anaconda是一个开源的用于科学计算的python发行版，其中包含了conda,python等180多个科学计算包及其依赖项，提供了包管理和环境管理功能，可以方便地解决多版本python并存、切换及第三方包安装更新问题。Anaconda使用工具conda来进行不同python environment及package的管理。

选择conda

相比与virtualenv等虚拟环境管理工具，conda管理实际的python环境，使用conda可以下载安装不同版本的python及在此环境下所需要用到的package，不同的python环境相互隔离，不会相互影响，每个环境都有独立版本的python解释器及独立的packages。conda使我们可以在不同的环境之间进行快速切换，并可以对每个环境及环境中的package进行安装、升级、删除。conda将几乎所有的工具、第三方包都当作package对待，包括python和conda自身。

使用conda进行环境管理

# 创建一个名为tensorflow的环境，指定python版本是3.6
conda create --name tensorflow python=3.6

# 安装好后，使用activate激活环境
activate tensorflow
source activate tensorflow # for Linux & Mac
# 激活后，会发现terminal多了tensorflow的字样，实际上，系统此时把PATH变更到当前python版本

# 此时，查看python版本，会发现已经切换到py3.6 
python --version

# 如果想返回默认的python环境
deactivate tensorflow
source deactivate tensorflow # for Linux & Mac
# 删除已有的python环境
conda remove --name tensorflow -all

用户安装不同的python环境都会被放在anconda目录的envs子目录下，或者使用

conda info -e

查看已安装的环境，当前被激活的环境会显示星号或括号

使用conda进行包管理

使用conda进行包管理与使用pip类似，若要安装tensorflow:

# 安装tensorflow
conda install tensorflow
# 查看已安装的packages
conda list

包管理常用操作：

# 查看某个指定环境的已安装包
conda list -n tensorflow

# 查找package信息
conda search numpy

# 安装package
conda install -n tensorflow numpy
# 若不用-n指定环境名，则被安装在当前激活的环境

# 更新package
conda update -n tensorflow numpy

# 删除package
conda remove -n tensorflow numpy

由于conda将conda、python都视为package，因此，可以使用conda来管理conda和python的版本，例如：

# 更新conda, 保持conda最新
conda update conda

# 更新anaconda
conda update anaconda

# 更新python
conda update python

由于anaconda.org服务器访问较慢，可以设置国内的镜像来下载安装package:

# 添加anaconda的tingshua镜像
conda config --add channels https://mirrors.tuna.tsinghua.edu.cn/anaconda/pkgs/free/

# 设置搜索时显示通道地址
conda config --set show_channel_urls yes
