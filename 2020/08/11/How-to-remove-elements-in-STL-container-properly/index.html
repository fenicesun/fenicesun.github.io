How to remove elements in STL container properly

2020-08-11 | programming

如何正确地删除STL容器中的元素


erase()函数的功能是用来删除STL容器中的元素，并释放空间。例：删除容器C中的某个元素C.erase(element)

remove()函数的功能：对于基于节点的容器(node-based container)，remove会做真正的删除操作；而对于连续内存容器(contiguous-memory container)，remove会将待删除元素移动到容器的尾部，但并不会减少容器的size，不涉及空间的释放。

STL中的容器按照内存分配方式可以分为连续内存容器和基于节点的容器。连续内存容器通常指vector/deque/string等在一块连续内存上保存元素的容器，而基于节点的容器通常指基于动态内存分配节点的容器，如list/slist等序列容器和set/map/multiset/multimap等关联容器

由于对于迭代器指向元素的删除，会导致迭代器位置不可用导致不确定行为，所以需要非常小心。无论如何都不要像下面这样删除元素：

Container<int> c;
for (auto iter = c.begin(); iter != c.end(); ++iter) &#123;
    if (badValue(*iter)) &#123;
        c.erase(iter);  // 可能导致不确定行为，别这样做
    &#125;
&#125;

正确的方法，推荐的方式用来删除元素，对于关联容器(map/set/multimap/multiset)：

Container<int> c;
for (auto iter = c.begin(); iter != c.end(); ++iter) &#123;
    if (badValue(*iter)) &#123;
        c.erase(iter++);  // 删除之前先获取了下一个迭代器位置的备份
    &#125; else &#123;
        iter++;
    &#125;
&#125;

而对于标准序列容器（vector/deque/string），则可以采取不同的策略。利用erase的返回值，一旦erase完成，它是指向紧随被删除元素的下一个元素的有效迭代器。

Container<int> c;
for (auto iter = c.begin(); iter != c.end(); ++iter) &#123;
    if (badValue(*iter)) &#123;
        iter = c.erase(iter);  // 通过erase方法的返回值来获取下一个元素的位置
    &#125; else &#123;
        iter++;
    &#125;
&#125;

list是特殊的，就遍历和删除来说，可以把list当作vector/deque这样的标准序列容器对待，也可以当作关联容器来对待，两种方式对list都适用。

其中对于特定对象和满足特定判别式的对象删除，推荐使用erase-remove/erase-remove_if的方式，并尽量使用区间成员函数代替for循环，以获取更高的效率和更清楚的意图，例如：

container.erase(remove(container.begin(),container.end(),value), container.end())
