原码、反码和补码


对于一个数字，计算机需要使用一定的可运算的编码方式对其进行编码。原码，反码，补码都是计算机对具体二进制数据的编码方式。

一个数字在计算机中二进制表示，叫做这个数的机器数。而机器数所要编码的实际数字，叫做真值。这里所要介绍的原码，反码和补码，就是将真值编码成机器数即二进制01序列的编码规则。

对于一个简单的四位数字7， 可以用四个二进制位0111这样的位序列表示。现实中计算机系统在数字的编码和运算时有几个问题：

  1. 除了要能表示整数，还需要能够表示负数，也就是机器数需要包含一个符号位（1表示负数，0表示正数）
  2. 计算机系统中，字长有容量和精度限制，通常使用固定长度的二进制位序列来表示一个数。（2字节short，4字节int，8字节long long等）
  3. 由于门电路进行减法运算较为复杂，编码方式在计数运算时需要能够将减法转换为加法。例如对于A - B必须转换为 A + ( -B)
  4. 与问题3相似，判断符号位的操作会使得硬件电路结构变得复杂，因此需要能将符号位一起参与到数字的运算当中去，并且只保留加法运算

原码、反码和补码的产生过程，就是为了解决引入符号位（正和负）和计算机做减法中的问题。

原码

原码即为符号位加上真值的绝对值，即用最高的一位表示符号（0为正，1为负），其余位表示数值。比如：（以8位二进制为例）

[+1] 原码 = 0000 0001
[-1] 原码 = 1000 0001

因为第一位是符号位，故8位二进制数的取值范围就是：[1111 1111] ~ [0111 1111]，即为[-127 ~ 127]。推广到n位整数，用原码表示的n位二进制数的取值范围就是：

-2^(n-1) ~ 2^(n-1)

原码就是我们通常意义上对二进制数的书面表示。原码是有缺陷的，对于8bit来说，它只能表示255个状态，因为[0000 0000](+0) 和 [1000 0000](-0)其实是一个数，造成了0这个真值的有两种编码的分歧。

让我们做一个原码表示数的运算。

0001 + 0010 = 0011 ( 1 + 2 = 3) // OK

0000 + 1000 = 1000 ( +0 + (-0) = -0 ) // May be reasonable

0001 + 1001 = 1010 ( 1 + (-1) = -2) // NOT OK

按二进制数的按位加法，只有正数和正数相加时不会出错，因为就是一个简单的二进制加法，因此原码通常用于正值的计算。而当正数与负数相加，或负数与负数相加，就会因为符号位的存在产生奇怪的结果，需要一套基于符号位判断的更复杂的运算规则才可以保证结果正确，而这对于硬件电路设计来说是不可接受的。于是，反码来了。

反码

如果用原码表示数字，让计算机辨别符号位对基础电路设计来说是不可接受的，而让符号位参与计算，显然对于减法（与负数相加）来说，结果是不正确的。这也是为何计算机内部不使用原码表示数字。为了解决原码做减法的问题，出现了反码。

我们知道，原码在符号位参与运算时会不准确，最严重的表现就是一个数加上他的相反数不等于零。例如：

0001 + 1001 = 1010 ( 1 + (-1) = -2)

0010 + 1010 = 1100 ( 2 + (-2) = -4)

反码的设计思想就冲着解决这一点，既然一个负数是一个正数的相反数，那干脆用一个正数按位取法来表示负数。具体的运算法则是：

正数的反码就是其原码

负数的补码在其原码的基础上，符号位不变，其余各位取反

例如：

[ +1 ] = [0000 0001] 原 = [0000 0001] 反

[ -1 ] = [1000 0001] 原 = [1111 1110] 反

[ +1 + (-1)] = [0000 0001] 反+ [1111 1110] 反 = [1111 1111] 反 = [1000 0000] 原 = -0

我们看到使用反码计算时，一个数与相反数相加不等于0的问题已经解决。但是似乎又引入了新的问题，让我们尝试两个负数相加：

[-1] = [1001] 原 = [1110] 反

[-3] = [1011] 原 = [1100] 反

[ -1 + (-3) ] = [1010] 反 = [-5] // Not equal

反码的运算解决了相反数问题，但是却让两个负数的相加出错了。回头想想计算机数字采用的编码方式需要解决的问题，核心就是要在带符号位的简单二进制相加计算中保证结果的正确性。而两个正数和两个负数的相加，其实都是一个加法问题，只是有无符号位的差别。而正数与负数相加才是真正的减法问题。

也就是说，只要正数与负数的加法不出错，那么问题就得到了解决。反码的负数相加出错，其实问题不大。我们只需要加实现两个负数加法时，将两个负数反码包括符号位全部按位取反相加，然后再给他的符号位强行置1就可以了。所以反码表示法其实已经解决了减法的问题，他不仅不会像原码那样出现两个相反数相加不为零的情况，而且对于任意的一个正数加负数，如： 0001（1）+1101（-2）=1110（-1） 计算结果是正确的。所以反码与原码比较，最大的优点，就在于解决了减法的问题。

那么为什么计算机科学家仍然不满足于使用反码呢？不知道你有没有注意到上面的一个运算：

[ +1 + (-1)] = [0000 0001] 反+ [1111 1110] 反 = [1111 1111] 反 = [1000 0000] 原 = -0

最大问题就是[1111 1111] 反 = -0, 而 -0 = 0？又导致了[1111 1111] = [0000 0000]，是不是和原码的那个问题似曾相识？好吧处女座，接下来介绍我们的大boss补码。

补码

提出补码，本质上是为了解决一个问题：有符号数原码由于符号位的存在导致的按位加权和与数值真值不对应导致的不能进行加减运算的问题。

将钟表想象成是一个1位的12进制数. 如果当前时间是6点, 我希望将时间设置成4点, 需要怎么做呢?我们可以:

  1. 往回拨2个小时: 6 - 2 = 4
  2. 往前拨10个小时: (6 + 10) mod 12 = 4
  3. 往前拨10+12=22个小时: (6+22) mod 12 =4

方法2和方法3中的mod是取模操作，16 mod 12 = 4， 即用16除以12后的余数是4。所以钟表往回拨（减法）的结果可以用往前拨（加法）代替！现在的焦点就落在了如何用一个正数，来替代一个负数。

同余

两个整数a, b，若它们除以整数m所得的余数相等，则称a，b对于模m同余。记作：a ≡ b (mod m)，举例说明：

4 mod 12 = 4

16 mod 12 = 4

28 mod 12 = 4

我们称4，16，28关于模12同余。

负数取模

正数进行mod运算很简单，但是负数如何取模呢？ 根据mod运算的数学定义：

x mod y = x - y * low_bound(x / y), for y != 0

上面公式的意思就是，x mod y等于x 减去 y乘上x与y商的下界。以-3 mod 2为例：

-3 mod 2

= -3 - 2 * low_bound(-3 / 2)

= -3 - 2 * low_bound(-1.5)

= -3 - 2 * (-2)

= -3 + 4 = 1

所以类似的可以得到：

(-2) mod 12 = 12-2=10

(-4) mod 12 = 12-4 = 8

(-5) mod 12 = 12 - 5 = 7

让我们继续回到时钟的问题上：

回拨2小时 = 前拨10小时
回拨4小时 = 前拨8小时
回拨5小时 = 前拨7小时

结合上面的同余概念，我们还可以发现：

-2 mod 12 = 10

10 mod 12 = 10

-2和10是同余的。想必你已经知道该怎么做了，那就是用同余的正数去代替负数。要实现这一步，还需要再介绍同余数的两个定理：(同余自反性和线性运算定理)

同余数自反性

a ≡ a (mod m)

线性运算定理

如果a ≡ b (mod m)，c ≡ d (mod m) 那么:

(1) a ± c ≡ b ± d (mod m)

(2) a * c ≡ b * d (mod m)

由这两个定理，我们可以得到：

7 ≡ 7 (mod 12)

(-2) ≡ 10 (mod 12)

7 - 2 ≡ (7 + 10) (mod 12)

现在我们为一个负数，找到了它的正数同余数。我们根据上面可以发现， 7 - 2的结果等于(7 + 10) mod 12，至此，我们成功将一个减法操作转换成加法与模运算。

在这里，我们再次强调原码，反码，补码的引入是为了解决做减法的问题。在原码，反码表示法中，我们把减法化为加法的思维是减去一个数，等于加上一个数的相反数，结果发现引入了符号位，却因为符号位造成了各种意向不到的问题。

但是从上面的例子中，我们可以看到其实减去一个数，对于数值有限制，有溢出的运算（模运算）来说，其实也相当于加上这个数的同余数。计算机的字长是定长的，即存储和处理的位数是有限的，因此它也有一个计量范围，即都存在一个“模”。事实上对于计算机中的存储的数值，通常都用限制长度的（2bytes,4bytes or 8bytes）位序列来存放。

也就是说，对于计算机中的存储的数值，我们可以不引入负数的概念，就可以利用同余数定理把减法当成加法来计算。补码的思想，就是在把减法当成加法时并不是必须要引入符号位。下面举一个例子，还可以顺便回答为什么负数的符号位是1，而不是0。

现在我们做一个四位二进制数的减法（先不引入符号位）

0110 (6) - 0010 (2) = 6 - 2 = 4 (计算机中没有减法器，所以无法运算)

这时候想想同余定理，减去一个数，可以等于加上这个负数的正同余数,从时钟运算中我们可以看出这个数与减数相加正好等于模。那么四位二进制数的模是多少呢？也就是四位二进制数的最大容量是2^4 = 16 = 10000b，那么-2的同余数就是14，既然如此

0110 (6) - 0010 (2) = 0110 (6) + 1110 (14) = 10100 (20 = 16 + 4)

OK，我们看到按照这种算法得出的结果是10100，但是对于四位二进制数，由于硬件限制只能存放四位，最高位将被舍去，低四位0100即是我们的计算结果（4）。

减去2，也就是加上-2，即加上-2和加上14得到的二进制结果是一样的，如果我们把1110（14）的最高位看作符号位后就是-2的补码，这可能就解释了为什么负数的符号位是1而不是0。在有符号的四位二进制数中，能表示的范围只有[-8 ~ 7]，而无符号数的作用和有符号数（-2）的作用效果其实是一样的。那正数的补码呢，就是他本身了。

到这里，我们发现原码、反码的问题，补码基本都解决了。而且补码中也不存在负零了，因为1000表示-8，0001（1）+ 1111 （-1）= 0000。负数与负数相加的问题也没了，1111（-1) + 1110 (-2) = 1101 (-3)

现在我们可以给出补码的求法（注意是求法）：

正数的补码就是他本身

负数的补码等于反码+1

也有另一种计算方法：原码自低位向高位，尾数的第一个‘1’及其右边的‘0’保持不变，左边的各位按位取反，符号位不变。这些都是补码的求法而已。当年学计组的时候，十分不理解为什么补码=反码+1，其实根本不用在意，只是补码恰好等于反码+1，忘掉它就是了。

最后我们再看看负数的补码为什么是反码+1。因为负数的反码加上这个负数的绝对值正好等于1111，再加1，就是1000，也就是四位二进制的模。而负数的补码是它的绝对值的同余数，可以通过模减去负数的绝对值，得到它的补码。所以补码就是它的反码+1。

总结

原码、反码和补码都是为了解决数字的二进制编码和运算中的问题。由于既要表示正数也要表示负数，原码引入符号位来表示数字的符号。但是由于符号位的引入，导致在有负数参与运算时结果不正确，且在0这个数字上出现双重编码。

因为原码的缺陷，计算机科学家们又发明了反码，反码解决了减法运算的问题，却在负数与负数的运算上出现了问题，且出现了-0这样的bug。

补码的出现彻底解决了减法与0的问题。它的核心是同余数运算，将一个减法转换成与同余数的加法，从本质上解决了由于符号位存在而导致的编码规则下按位加权的加减运算和真值运算结果不对应的问题，实现了加减运算的完美统一。 现代计算机中都采用补码作为内部数值编码方式。

由于新冠肺炎疫情的影响，年后一直居家办公。耗时一周多，终于利用每晚的空余时间把这本买来已久的《霍乱时期的爱情》读完。加西亚-马尔克斯以其对生活无比敏感的洞察力，叙述了一个完全不魔幻、相当写实的、跨越了半个多世纪的浩瀚漫长的爱情故事，并通过小说展现了对人生、爱情、衰老和死亡的思考。毫不恭维地说，这是我所读过的最伟大的爱情小说。

小说内容横跨了五十三年七个月零十一天的琐碎中三个主人公大半部分的人生，主线情节并不复杂，但马尔克斯写得太过真实，他笔下的爱情，就是生活中我们自己也看不清、也看不懂的情感，让人读完感受太多太复杂。

开篇用了倒叙手法。城中受人尊重的胡维纳尔-乌尔比诺医生已经八十三岁高龄，在暮年的生活中正感觉自己逐渐被衰老侵蚀。在他的棋友赫雷米亚-德圣阿莫尔因为“衰老恐惧症”自杀而亡的第二天，他便因为爬上梯子捉一只鹦鹉不慎失足摔死。临死前，胡维纳尔-乌尔比诺医生用尽最后一丝力气，向并不算和睦的妻子费尔明娜-达萨说：“只有上帝才知道我有多爱你”。医生在霍乱时期为城市的卫生事业做了很大的贡献，倍受这个城市的人推崇，人们为他举行了盛大的葬礼仪式。吊唁过后的当晚，弗诺伦蒂诺-阿里萨出现，并重申对费尔明娜-达萨半个多世纪以来永恒的忠诚和不渝的爱情。费尔明娜-达萨愤怒地赶走了他，悲伤的她为了死去丈夫哭泣惋惜的同时，更多想起的却是弗洛伦蒂诺-阿里萨。

这时候小说把主线切到了故事的最开始。弗洛伦蒂诺-阿里萨是一个私生子，母亲是一个勤劳的杂货铺店主。父亲死后，阿里萨辍学到了邮电局当了一个电报员，他身材矮小、衣衫简陋，喜欢朗诵诗歌和拉小提琴。天真的生活在他遇到费尔明娜-达萨后结束，他在送电报的时候偶然遇到了费尔明娜-达萨，对她一见钟情。阿里萨每天都在费尔明娜-达萨上学时，坐在她必经之路上的小花园里假装看书，进行隐秘的追求。费尔明娜-达萨几年前和姑妈跟着父亲从外地来到这座城市，父亲洛伦索-达萨传言是一个骡子商人，暴躁粗鲁，对女儿的管教十分严苛。费尔明娜也注意到了阿里萨，并对他产生了好奇。终于，阿里萨鼓起勇气给费尔明娜送了一封情书，两人便暗地里开始了情书通信，为了躲开父亲，信都是由姑妈帮忙递送或者在前一封信中约好下一封信的藏匿地点。就这样通信了两年，一次费尔明娜在课堂上写情书时被修女校长抓到，由于拒绝说出恋人是谁，被教会学校开除。父亲洛伦索-达萨暴怒，把她的姑妈赶回了老家，并找到阿里萨对峙，申明了他在费尔明娜母亲死后唯一的愿望就是希望女儿能成为一位令人尊敬的夫人，让阿里萨远离自己的女儿。

为了让女儿忘掉这个这段恋情，洛伦索-达萨决定强行带费尔明娜外出旅行。他们跟着骡队出发，一路的艰辛让费尔明娜苦不堪言，甚至累到无暇去想阿里萨。他们到费尔明娜母亲的娘家亲戚家住了一年多，在此期间，由于探听到旅程路线，阿里萨通过路线各地的电报员同事，总能保持着和费尔明娜的通信。费尔明娜-达萨在与自己的表姐妹相处中也变得成熟，而且知道了一个让她吃惊的道理：婚姻不需要爱情也可以幸福。

两年的旅途终于结束，回到家里。一天在喧闹的市场中，弗洛伦蒂诺-阿里萨偶遇了费尔明娜-达萨。费尔明娜-达萨转头看见阿里萨那冰冷的眼睛、青紫色的面庞和因爱情的恐惧而变得僵硬的双唇。忽然坠入了失望的深渊，在那一瞬间，她恍然大悟，原来自己对自己撒了一个弥天大谎。这是一次大型网恋奔现失败现场，费尔明娜意识到自己想象中的他不过是个影子，和他并不是一个人。

“今天，见到您时，我发现我们之间不过是一场幻觉。”

出身当地名门望族乌尔比诺-德拉卡列家族的胡维纳尔-乌尔比诺医生从巴黎学成归来，一次偶然的机会，费尔明娜-达萨肠胃不适被当成了霍乱，请来胡维纳尔-乌尔比诺医生看病。医生被费尔明娜所吸引，开始了热烈的追求。费尔明娜并不喜欢医生，医生便先后通过她父亲洛伦索-达萨、教会的修女轮番来做思想工作，都被费尔明娜拒绝。在医生热烈追求时，费尔明娜甚至还不断收到城里仰慕医生的女士们送来的匿名威胁信，这让她提心吊胆。费尔明娜在发现自己二十一岁迫在眉睫时慌了手脚，害怕机会稍纵即逝，二十一岁是她心里向命运屈服的秘密界限。在感受到医生和表姐的调情带来的嫉妒后，接受了医生。两人快要结婚的消息传来，弗洛伦蒂诺-阿里萨消沉不已，无法振作。

从费尔明娜-达萨和胡维纳尔-乌尔比诺医生结婚开始，小说的主线开始分叉，分别从费尔明娜-达萨和弗洛伦蒂诺-阿里萨两个支线穿插描述两个不同的生活，直到乌尔比诺医生去世，两个支线叙述才最终合并回主线。

费尔明娜-达萨和胡维纳尔-乌尔比诺夫妇在欧洲度过了两年的蜜月。刚开始，费尔明娜并不喜欢医生，甚至有些抗拒，医生也明白自己并不爱她，同她结婚是因为喜欢她的高傲严肃，她的力量，也因为自己的一点儿虚荣心，然而随着蜜月之旅的推进，渐渐地他们之间似乎开始建立了爱情。他们在巴黎到处游逛，买了很多的衣服、家具，每天起床后做爱到中午，拜访欧洲的名流。但当回到回到家乡，别人问起她欧洲之旅有何感受，她只是淡淡地说了四个字：“浮华而已”。于此同时，弗洛伦蒂诺-阿里萨在一艘内河船的疗伤之旅中被最后也不知道是谁的女人给强奸夺取童真。回到家后，它的母亲特兰西多-阿里萨为了治愈儿子的相思病，故意把因战争被炸毁房子而前来投奔的以色列寡妇安置到了弗诺洛蒂诺-阿里萨的房间，两个人疯狂做爱，放肆地爱恋了六个月。阿里萨自信已经从费尔明娜-达萨之痛中恢复了过来，直到一个星期日，他在没有任何预感的情况下看见从欧洲蜜月回来的费尔明娜，她挽着丈夫的手臂，已经有了身孕。她比以往更美、更年轻，却也比以往任何时候都遥不可及，和丈夫组成了令人羡慕的一对。弗诺伦蒂诺-阿里萨没有妒忌，也没有愤怒，而是感到一种巨大的自卑。他觉得自己可怜、丑陋、低贱，不仅配不上她，也配不上世界上任何一个女人。

弗诺伦蒂诺-阿里萨看见光彩照人的费尔明娜后，决心要赢得名誉和财富以配得上她，同时他又认定，胡维纳尔-乌尔比诺医生是会死的。他在心中暗暗起誓，要等到医生死去的那天。他找到自己的叔叔，加勒比河运公司董事长莱昂十二，向他请求一分差事。阿里萨从抄写文书的书记员的岗位勤奋努力地干活，三十年间，做过了所有的岗位，了解了公司运作的每一项秘密，可他仍旧写不好哪怕一篇商业文书，莱昂十二训斥他总是把商业文书写成了情书。弗洛伦蒂诺-阿里萨无法摆脱抒情体，因为他时时刻刻都在思念费尔明娜-达萨。为了减轻费尔明娜之痛，他开始了长达三十年的令人三观尽毁的荒唐猎艳生涯，企图以和其他女人的床上关系来缓解对费尔明娜-达萨的思念。和以色列寡妇的那段经历为他打开了街头爱情之门。此后很多年，他一直都在夜间出动，和不同类型的女人发生着性关系，无论是女仆、黑女人还是寡妇，他都照单全收，小说的这部分描写多到可以单独抽出来写一本《阿里萨猎艳记》。到了后来，他已经说不清这通奸习惯到底是出于内心需要，还是单纯的身体恶习。他总是在深夜从后门溜进情妇家中，天亮前不久再踮着脚逃走，他对此的想法是怕费尔明娜知道自己对她的不忠。整个人陷在一个矛盾里，不停地想努力摆脱自己所受的奴役，却又不想让自己的背叛给她带去痛苦。阿里萨甚至还把他所经历过的女人记在了本子上，到小说最后发生过关系的女人多达622个。三十年的时间里，他在加勒比河运公司步步高升，直到坐到仅次于董事长莱昂十二的位置，人生地位终于实现了改变。

费尔明娜在两年的幸福蜜月回家之后，发现生活的考验才刚刚开始。他那刻薄的婆婆和愚昧陈腐的小姑子们让她绝望，她用尽全力承担女主人的职责，照顾起一家人的饮食起居，可收到的只有无休止的挑剔和批评，对她父亲不正当营生的讽刺。甚至做梦都逃不过婆婆的指责，觉得自己已经变成了一个雍容华贵的女仆。更糟糕的是，她的丈夫完全指望不上，乌尔比诺医生甘愿屈从于家族礼教，对她的需求置若罔闻，只会找各种宏大的理由来为自己辩解。此刻她才怀疑，在职业权威和世俗的迷人外表下，她嫁的这个男人其实是个无可救药的懦夫：一个靠姓氏带来的社会地位而耀武扬威的可怜虫。这对夫妻更为荒谬的是，在那段不幸的岁月里，他们在公众面前却表现的无比幸福。终于有天费尔明娜忍无可忍，坚持要离家出走，丈夫不得已带着她去欧洲散心。又逗留了两年，费尔明娜接到婆婆布兰卡夫人去世的电报，他们又一次返回家中，并且怀上了第二个孩子。从此以后，直到丈夫出轨林奇小姐前，费尔明娜度过了人生最幸福的三十年，她把自己的爱寄托给了孩子，对于丈夫，那也是他们相爱的最美好的时期，不慌不忙，适宜得体。费尔明娜曾经跟朋友表示：如果再来一次，她还会选择现在的丈夫。在新生活的全胜时期，弗诺伦蒂诺-阿里萨在加勒比河运公司也扶摇直上，就在费尔明娜以为已经把他从记忆中彻底抹掉的时候，他又成为了她怀旧思绪中的一个幽灵，总觉得生活中发生了什么不可弥补的事情。

三十年的平静生活过后，胡维纳尔-乌尔比诺医生在一次出诊中出轨了芭芭拉-林奇小姐，费尔明娜舒心的生活又一次被打破，这次她愤怒难平，又一次地离家出走，去了表姐的庄园，就像在这座城市消失了一样。两年后，不得已的医生只得亲自把费尔明娜接了回来。费尔明娜和医生琐碎的生活中的争执，积攒了太多怨恨，后来她回忆他们的一生时，想到的更多是挫折，而非满足。当费尔明娜向丈夫喊道:你就没有发现我一点也不幸福吗？但是医生只是说道：

你要记住，一对恩爱的夫妻，最重要的不是幸福，而是稳定

这时弗洛伦蒂诺-阿里萨已经快50岁了，母亲已经去世，他接替了莱昂十二成为了加勒比河运公司的董事长。在日常的生活中时刻感觉到时间的消逝和衰老的来临，自从养鸽女和他偷情被丈夫砍杀的意外之后，他的街头爱情也愈来愈少，只剩下几个固定炮友。一个下午，当他从最久远的往事开始回忆自己的过往，回顾了一桩桩猎艳情事，以及种种数不清的往事，而这一切皆由他发誓要让费尔明娜-达萨属于他而起，可直到这一刻，他才发现自己的一生几乎都要过去了，已经越来越明显地感觉到身体在走下坡路，而当他有一次在电影院偶遇费尔明娜，竟然发现她也挡不住岁月的侵蚀，暗自觉得自己可能要败给时间。而同样是三十年，对于费尔明娜-达萨来说，那是她一生最快乐最舒心的三十年。

胡维纳尔-乌尔比诺医生死后，小说剧情主线在这里合并。已经76岁的弗洛伦蒂诺-阿里萨等了半个世纪，终于等到了这一天，他在医生葬礼之后向费尔明娜-达萨重申自己的爱情，觉得他刚刚杀死了围困自己半个多世纪的老虎。 阿里萨占据了费尔明娜的脑海，这让她怒火中烧，越想越愤怒，最终写了一封可怕的辱骂信。阿里萨回了一封他有史以来写得最像商业信函的信，信中没有提起过去的一切，只是写下了一些对人生、爱情、老年和死亡的思考。这封信反而慰藉了守寡生活中快要疯掉的费尔明娜，于是他们开始了暮年的联络。五十一年的守候，622位情人的洗礼，终于让他成长为她需要、可以携手走向死亡的人。反过来，他也需要她帮忙解开这个从少年起困扰了半个多世界的执念。 这时报社出于某些目的集中攻击当地贵族家庭，爆出了当年费尔明娜父亲洛伦索-达萨所做出的丧尽天良的勾当和它的亡夫乌尔比诺医生生前和她闺蜜的种种私通行为。费尔明娜-达萨受到了巨大的打击，在阿里萨的陪伴下才恢复过来。费尔明娜那个像她婆婆一样的女儿极力反对费尔明娜和阿里萨来往，她说“我们中年的爱情已属荒唐，老年的爱情就是卑鄙”，这激怒了费尔明娜-达萨，她赶走了女儿，对儿媳感叹到：“一个世纪前，人们毁掉了我和这个可怜男人的生活，因为我们太年轻；现在，他们又想在我们身上故技重施，因为我们太老了。”

因为父亲所受的侮辱，对死去丈夫的怨恨，对自己闺蜜的虚伪的愤怒，和女儿的吵架，费尔明娜达萨最终接受了阿里萨坐轮船散心旅行的邀请。两个孤独的人时刻半个多世纪以后终于坐在了一起，河岸的景色早已变化，整个世界都变了，两个人早已到了古稀之年，满身都是衰老带来的痕迹。而他们终于互相了解，互相安慰，甚至还做了几次爱。像一对经历了生活磨练的老夫老妻，超越了激情的陷井，超越了幻想无情嘲弄和醒悟的海市蜃楼：超越了爱情。因为他们在一起生活的几天，足以发现无论何时何地，爱情始终是爱情，只不过距离死亡越近，爱就越浓郁。轮船之旅快要结束回到港口的时候，费尔明娜不愿回到以往的生活。阿里萨让船长挂上霍乱旗帜，让船在两地之间不停的往返而不用靠岸。

以上就是小说的主要剧情。霍乱在拉丁美洲的母语西班牙语中有狂热的意思，预示着狂热的爱情。在小说中，霍乱又成为费尔明娜认识医生以及最后船上升起霍乱旗帜的线索。看完这本书，其实感受十分复杂。费尔明娜和阿里萨这条主线上，将各种人物经历的丰富多样的爱情串联起来，有忠贞的、隐秘的、柏拉图式的、放荡的、转瞬即逝的...... 如果说费尔明娜和乌尔比诺夫妇的一生让我知道了婚姻是责任和包容，但看完我还是没能明白这里爱情是什么。

其实仔细回想，不明白就对了。马尔克斯写了这么长一篇你厌恶和欲罢不能的事，再给你泼一盆人生来孤独的凉水，就是告诉你： 哪有什么爱情。要么是顽固不化的执念，要么是转瞬即逝的冲动，要么是本质为亲情的婚姻。书里这些我们自己也看不清、看不懂的欲望、冲动和感情，打着爱情的旗号，把两个素不相识的人带进一种自私的、不健康的依赖关系中。这本书里的每一种爱情到最后，都会觉得所有的爱情都是爱自己。

长久以来，我们给“爱情”过于美好的设定，它必须纯洁坚贞、天长地久、矢志不渝。但这世界上的爱情本来就是一万种模样，很多时候它是混乱的、曲折的、痛苦的、如霍乱一般狂热的，但是那从来不妨碍一个人对另一个人说：我爱你，一生一世。

信息的表示

在计算机中，信息就是 位+上下文。我们所见到的文本文件，打开后会看到有很多的字符，在现代系统中，通常使用ASCII标准或者Unicode标准对常见的字符进行编码，你所见到的每个字母或者符号，都被映射到一个具体的编码上，这个编码在ASCII标准中是一个由7个二进制位表示的整数，它的范围是0 - 127（2^7 -1，共128个），即ASCII编码0 - 127 都表示了一个唯一对应的字符，它覆盖了大小写字母，数字，以及键盘上的绝大多数符号。

那么既然ASCII码的范围只有0-127，那计算机是如何表示成千上万的汉字，斯拉夫文字或者日语韩文呢？上面说到ASCII用了7个二进制位来表示一个字符，那么这里就需要扩展编码的长度，用8个二进制位就可以表示256个字符，用16个二进制位就可以表示65536个字符。这样绝大多数日常使用的汉字也可以被编码出来。但是相应的，原来用7个位就能存一个字符，现在得用16个位，这就消耗了更多的存储的空间。

现在我们知道，文本文件也是通过二进制位来进行存储。这里有个基本思想，系统中的所有信息--包括磁盘文件、存储器中的程序指令、存储器中放置的用户数据以及网络中传输的数据，都是由一串二进制位来表示的。但如何知道这一串二进制位的序列究竟存储的是什么呢？区分不同的数据对象的唯一方法就是我们读取到这些数据时的上下文。不同的上下文中，一个同样的字节序列可能表示为一个整数、浮点数、字符串或者机器指令。


程序如何被编译出来执行

unix系统中，C程序通过编译器由一堆记录代码的文本文件(源文件)，最终被其他程序转化为一系列的低级机器语言指令，然后这些指令按照一种称为可执行目标程序的格式打包，并以二进制文件的形式存储在磁盘上。目标程序也称为可执行目标文件。

让我们看一看经典的hello.c

#include <stdio.h>

int
main() &#123;
    printf("Hello World\n");
    return 0;
&#125;

而从源文件到目标文件的转换是通过编译器驱动程序来完成，具体工具则是GNU项目里的gcc

unix> gcc -o hello hello.c

在这里，GCC编译器读取源程序文件hello.c，并把它翻译成一个可执行目标文件hello。这个翻译过程分为四个阶段，如下图。（预处理器、编译器、汇编器和链接器）一起构成了编译系统(compilation system)

  • 预处理阶段

    预处理器(cpp)根据以字符#开头的命令，修改原始的源程序，将include及define的实际源代码/数据对#指令进行文本替换，插入到程序文本中得到另一个c程序，以.i作为扩展名。

  • 编译阶段

    编译器(ccl)将文本文件hello.i翻译成文本文件hello.s，这是一个汇编语言程序。汇编语言程序中每条语句都以一种标准的文本格式表述了一条低级机器语言指令。

  • 汇编阶段

    汇编器(as)将hello.s翻译成机器语言指令，即0101...的二进制位码，并把这些指令打包成可重定向目标程序(relocatable object program)的格式，并将结果保存为目标文件hello.o。hello.o是一个二进制文件，他的字节编码是机器语言指令。

  • 链接阶段

    连接器(ld)将hello.o目标文件与hello.c这个程序调用到的库函数目标文件进行合并，得到最终hello可执行文件。上面hello.c程序调用了c标准库里的printf函数，因此ld会将printf.o与我们的hello.o合并得到hello，这里的hello就是一个可执行目标文件(可执行文件)，可以被加载到内存中，由操作系统执行。


处理器是如何执行程序的

我们已经写好并编译出了一个可执行目标程序hello，并存在了磁盘上。现在想要运行这个可执行文件，在unix上当然就是打开shell，输入可执行目标程序的名称。shell将这个hello加载并运行，然后等待程序运行终止。hello运行后在屏幕上输出了Hello World的信息。

unix> ./hello
Hello World
unix>

系统的硬件组成

为了理解运行hello程序时发生了什么，需要了解一个典型的系统的硬件组织

  1. 总线

    贯穿整个系统的一组电子管道，它携带信息字节并负责在各个部件中传递。通常总线被设计成传送定长的字节块，也就是字(word)。字中的字节数（即字长）是一个基本的系统参数，在各个系统中的情况不尽相同。32位机器的字长是4个字节，64位机器的字长是8个字节。

  2. I/O设备

    输入/输出（I/O）设备是系统与外部世界的联系通道。每个I/O设备都通过一个控制器或适配器与I/O总线相连。控制器和适配器的区别主要在于他们的封装方式。控制器是置于I/O设备本身的或者系统主板上的芯片组，而适配器则是插在主板卡槽上的卡（PCIE）。他们的功能都是在I/O总线和I/O设备之间传递信息。

  3. 主存

    主存是一个临时的存储设备，在处理器执行程序时，用来存放程序指令和程序处理的数据。物理上，主存是一组动态随机存储器(DRAM)组成的阵列，CPU通过地址线的输出告诉存储器要读取哪一个地址的数据，地址线的数量决定了存储器中的寻址范围。从逻辑上来说，存储器是一个线性的字节数组，每个字节都有其唯一的地址（数组索引）。

  4. 处理器

    CPU是解释执行存储在主存中指令的引擎。处理器的核心是一个字长的寄存器，称为程序计数器（PC）。PC中永远都存储着下一条机器指令的地址。

    系统通电后，直到系统断电为止，处理器一直在不断的执行程序计数器所指向的指令，并更新程序计数器，使其指向下一条指令。

    指令的操作由指令集在硬件层面来定义，其操作围绕着主存，寄存器和算术/逻辑单元（ALU）进行。寄存器是一个小的存储设备，由一些1字长的寄存器组成，每个寄存器都有唯一的名字（EAX、EBX等）。ALU进行数据算术/逻辑的运算，由加法器、移位器等模块组成。

    cpu可能执行的一些操作：

    * 加载：把一个字从主存复制到寄存器，覆盖原寄存器的内容 * 存储：把一个字从寄存器复制到主存的某个位置，覆盖这个位置的内容 * 操作：把两个寄存器的内容复制到ALU，ALU对这两个字做算术操作，并将结果存放到一个寄存器中（累加器） * 跳转：修改程序计数器中的指令，使其跳转执行其他的指令

    两个概念：

    + 指令集结构：描述每条机器代码指令的效果； + 微体系结构：描述处理器实际的硬件实现；
系统如何运行hello程序

初始时，我们在键盘上敲击命令，每次敲击操作触发中断处理程序，将我们的字符读入寄存器，再放入到存储器中

当在键盘上敲回车键时，shell执行一系列指令来加载可执行的hello文件，将hello目标文件中的代码和数据从磁盘拷贝复制到主存，处理器开始执行hello程序中的机器语言指令。这些指令将Hello World字符串的字节从主存复制到寄存器，再从寄存器复制到显示设备，最终显示在屏幕上。


存储器层次结构

在程序运行的过程中，系统花费了大量的时间把信息从一个地方挪到另一个地方。程序加载时，它们被从磁盘复制到主存，处理器执行时，指令从主存复制到处理器，最后复制到显示设备。从编程的角度，这样的复制就是开销。系统设计的一个主要目标就是使这些复制操作尽快完成。

针对这种频繁的挪动操作，系统设计者在处理器和主存之间采用了更小、更快的存储设备，即高速缓存。作为暂时的集结区域，用来存放处理器近期可能会需要的信息。L1级别的高速缓存访问速度几乎和访问寄存器一样快，L2级别的高速缓存访问时间大概比L1慢5倍，但仍然比主存快10倍。L1和L2高速缓存采用静态随机访问存储器硬件实现（SRAM），成本高于DRAM。

有了高速缓存以后，处理器和又大又慢的设备（主存/磁盘）中被塞入了几级更快更小的存储设备，这就形成了一个存储器层次结构。这个结构从L0(寄存器) -> L1(L1 高速缓存) -> L2(L2高速缓存) -> L3(L3高速缓存) -> 主存（DRAM）-> 磁盘 ，从上到下，访问速度越来越慢，容量越来越大。存储器层次结构的主要思想是上一层的存储器作为下一层存储器的高速缓存。因此，寄存器就是L1的高速缓存，L1是L2的高速缓存，L2是L3的高速缓存，L3是主存的高速缓存，主存又是磁盘的高速缓存。


操作系统

应用程序不会直接访问键盘、显示器、磁盘或者主存，取而代之的是，他们依靠操作系统提供服务。我们把操作系统看作是应用程序和硬件之间插入的一层软件。所有应用程序对硬件的操作都必须通过操作系统。

操作系统有两个基本功能：

  1. 防止硬件被失控的应用程序滥用；
  2. 向应用程序提供简单一致的机制来控制复杂而又通常大相径庭的设备；

操作系统通过几个基本的抽象概念（进程、虚拟存储器和文件）来实现这两个功能。文件是对I/O设备的抽象表示，虚拟存储器是对主存和磁盘I/O的抽象表示，进程则是对处理器、主存和I/O设备的抽象表示。

进程

进程是操作系统对正在运行的程序的一种抽象。在一个系统中可以同时运行多个进程，而每个进程看上去可以独占处理器、主存和I/O设备。传统的单处理器系统在一个时刻只能运行一个程序，而多核处理器真正同时可以执行多个程序。无论在单核还是多核系统中，一个CPU看上去都像在并发地执行多个进程，这是通过处理器在进程间切换来实现的。操作系统实现这种交错执行的机制称为上下文切换。操作系统保持跟踪进程运行所需的所有状态信息，这种状态就是上下文。包括PC和寄存器的当前值，以及主存的部分内容。任何一个时刻单处理器系统都只能执行一个进程的代码。当操作系统决定要把控制权从当前进程转移到某个新进程时，就会进行上下文切换，即保存当前进程的上下文，加载新进程的上下文，然后传递控制权到新进程。

线程

一个进程可以有多个称为线程的执行单元组成，每个线程都运行在进程的上下文中，并共享同样的代码和全局数据。

虚拟存储器

虚拟存储器是一个抽象的概念，为每个进程提供一个假象，即每个进程都在独占使用主存。每个进程看到的是一致的存储器，称为虚拟地址空间。虚拟地址空间有由大量准确定义的区构成，每个区有专门的功能。从低地址到高地址包括：

  1. 程序代码和全局变量
  2. 堆
  3. 共享库映射区域
  4. 栈
  5. 内核虚拟存储器
文件

如开篇所述，文件就是字节序列，仅此而已。每个I/O设备，包括磁盘、键盘、鼠标、显示器、甚至网络，都可以视为文件。


重要主题

并发和并行

并发(concurrency)是一个通用的概念，指一个同时具有多个活动的系统。是多个程序可以同时运行的现象，系统具备处理多个任务的能力，但不一定是同时处理，可能round-robin。而并行(parallelism)指的是同一时刻处理多个任务，必须具有同时处理多任务的能力。按照系统层次结构中由高到低的顺序有三个层次的并发：

  1. 线程级并发

    使用线程，可以在一个进程中执行多个控制流。在单处理器时代，并发只是使用分时技术模拟出来的效果，多个任务切换运行，看上去有同时执行的效果。而多处理器和超线程技术出现后，有了真正的并行支持。多处理器系统是将多个CPU核集成到一个集成电路上，每个核有自己的L1和L2高速缓存，并同时共享更高层次的L3高速缓存，以及到主存的地址线接口。超线程是一项允许一个CPU核执行多个控制流的技术。它涉及CPU某些硬件有多个备份，比如程序计数器和寄存器，而其他电路结构则共享，超线程处理器可以在单个周期的基础上决定要执行哪一线程，使CPU能够更好地利用它的资源。假设一个线程必须等待某些数据被装载到高速缓存中，那CPU可以继续执行另一线程。

  2. 指令级并发

    在较低的抽象层次上，同时执行多条指令的属性称为指令集并行。通过流水线技术，将一条指令所需要的活动划分为不同的步骤，将处理器的硬件组织为一系列的阶段，每个阶段执行一个步骤。这些阶段可以并行的操作，用来处理不同指令的不同部分。如果处理器可以达到比一个周期一条指令更快的操作，就称之为超标量（superscalar）处理器。

  3. 单指令、多数据并行

    在最低层次上，许多现代处理器拥有特殊的硬件，允许一条指令产生多个可以并行执行的操作，这种方式称为单指令、多数据，即SIMD并行。

抽象在计算机系统中的重要性

抽象是计算机科学最为重要的概念之一。在处理器里，指令集结构提供了对实际硬件的抽象，使用这个抽象，机器代码程序表现的好像是运行在一个一次只执行一条指令的处理器上。文件是对I/O的抽象，虚拟存储器是对程序存储器的抽象，而进程是对一个正在运行的程序的抽象。虚拟机是对整个计算机（包括操作系统、处理器和内存）的抽象。

GCC常用编译选项


  1. -c：只激活预处理,编译,和汇编,也就生成obj文件
  2. -S：只激活预处理和编译，就是指把文档编译成为汇编代码。
  3. -E：只激活预处理，不生成文档，需要把他重定向到一个输出文档里。
  4. -o：定制目标名称，缺省的时候gcc 编译出来的文档是a.out
  5. -ansi：关闭gnu c中和ansi c不兼容的特性，激活ansi c的专有特性。
  6. -Dmacro：相当于C语言中的#define macro
  7. -Dmacro=defn：相当于C语言中的#define macro=defn
  8. -Umacro ：相当于C语言中的#undef macro
  9. -Idir：指定头文件路径。
  10. -llibrary：指定库
  11. -Ldir：定制编译的时候，搜索库的路径。
  12. -g：指示编译器，在编译的时候，产生调试信息。
  13. -static：此选项将禁止使用动态库，所以，编译出来的东西，一般都很大。
  14. -share：此选项将尽量使用动态库，所以生成文档比较小，但是需要系统由动态库。
  15. -O0 -O1 -O2 -O3：编译器的优化选项的4个级别，-O0表示没有优化,-O1为缺省值，-O3优化级别最高
  16. -Wall：会打开一些很有用的警告选项，建议编译时加此选项。
  17. -std：指定C标准，如-std=c99使用c99标准，-std=gnu99，使用C99 再加上 GNU 的一些扩展。

ListView

SimpleAdapter

SimpleAdapter用来将自定义数据绑定到视图

SimpleAdapter(Context context, List<? extends Map<String, ?>> data, int resource, String[] from, int[] to)
  • Context context:上下文(当前的Activity)
  • List> data:一个嵌套Map集合类型的list集合数据源
  • int resource:这个是你设定用于接收Map-Object类型的Activity,这个布局中必须包括了to中定义的控件id
  • String[] from:第四个参数通过源码可以看出需要的是一个K值的字符串数组
