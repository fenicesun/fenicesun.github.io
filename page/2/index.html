使用Anaconda来管理python环境

Anaconda是一个开源的用于科学计算的python发行版，其中包含了conda,python等180多个科学计算包及其依赖项，提供了包管理和环境管理功能，可以方便地解决多版本python并存、切换及第三方包安装更新问题。Anaconda使用工具conda来进行不同python environment及package的管理。

选择conda

相比与virtualenv等虚拟环境管理工具，conda管理实际的python环境，使用conda可以下载安装不同版本的python及在此环境下所需要用到的package，不同的python环境相互隔离，不会相互影响，每个环境都有独立版本的python解释器及独立的packages。conda使我们可以在不同的环境之间进行快速切换，并可以对每个环境及环境中的package进行安装、升级、删除。conda将几乎所有的工具、第三方包都当作package对待，包括python和conda自身。

使用conda进行环境管理

# 创建一个名为tensorflow的环境，指定python版本是3.6
conda create --name tensorflow python=3.6

# 安装好后，使用activate激活环境
activate tensorflow
source activate tensorflow # for Linux & Mac
# 激活后，会发现terminal多了tensorflow的字样，实际上，系统此时把PATH变更到当前python版本

# 此时，查看python版本，会发现已经切换到py3.6 
python --version

# 如果想返回默认的python环境
deactivate tensorflow
source deactivate tensorflow # for Linux & Mac
# 删除已有的python环境
conda remove --name tensorflow -all

用户安装不同的python环境都会被放在anconda目录的envs子目录下，或者使用

conda info -e

查看已安装的环境，当前被激活的环境会显示星号或括号

使用conda进行包管理

使用conda进行包管理与使用pip类似，若要安装tensorflow:

# 安装tensorflow
conda install tensorflow
# 查看已安装的packages
conda list

包管理常用操作：

# 查看某个指定环境的已安装包
conda list -n tensorflow

# 查找package信息
conda search numpy

# 安装package
conda install -n tensorflow numpy
# 若不用-n指定环境名，则被安装在当前激活的环境

# 更新package
conda update -n tensorflow numpy

# 删除package
conda remove -n tensorflow numpy

由于conda将conda、python都视为package，因此，可以使用conda来管理conda和python的版本，例如：

# 更新conda, 保持conda最新
conda update conda

# 更新anaconda
conda update anaconda

# 更新python
conda update python

由于anaconda.org服务器访问较慢，可以设置国内的镜像来下载安装package:

# 添加anaconda的tingshua镜像
conda config --add channels https://mirrors.tuna.tsinghua.edu.cn/anaconda/pkgs/free/

# 设置搜索时显示通道地址
conda config --set show_channel_urls yes

The principle of variable naming

Posted on 2016-11-05 Edited on 2019-11-05 In programming

变量命名的原则

类型和变量的命名绝对是平时开发中保证代码整洁、可读性和可维护性的重要环节，每次在新系统的开发中，针对变量、函数和类的命名我都要反复斟酌，以获得最高程度的合理性，下面介绍一些《代码整洁之道》中的变量命名principle

名副其实

名字一定要符合变量的实际含义，体现变量使用的目的。尽量不要使用单字母变量

避免误导

  • 不应该使用现实世界中已有的专有名称，如hp,aix,sco
  • 不应该使用两组差异性太小的单词，很容易造成阅读及使用混乱
  • 变量名中不应该包含类型信息

做有意义的区分

  • 不要使用字母加数字的命名a1,a2,a3…，而应该根据其意义命名
  • 命名中不用加入废话，比如Product好于ProductData/ProductInfo，不需要添加a,an,the等定冠词

不要创造单词


变量需要易于搜索

单字母变量很难在大型工程代码中搜索到想要的代码块，因此除非极短方法中的局部变量，其余都不应该使用单字母进行变量命名。变量必须很容易在工程目录中搜索区分出来。

避免使用编码

  • 不要在名称中加入类型信息，比如phone_number，不要叫phone_number_str
  • 成员变量不需要加m_前缀，可以使用_后缀

隐性约定

  • 类名应该是名词或名词短语， 如Customer,WikiPage,Account,AddressParser
  • 方法名应该是动词或动词短语，如postPayment, deletePage, save, get, set

概念的一致性

同一个概念需要在代码中保持同一个单词，即概念一致性。比如取数据的操作，要么都叫fetch，要么都叫get。同样的，一堆代码中不能既有controller,又有manager还有driver。DeviceManager和ProtocolController之间没有本质区别。

不用双关语

还是一致性的问题，同一个单词不可以在一个代码项目中体现两个不同的概念，其体现的意义必须一致。

使用解决方案领域名称

尽量使用工程所涉及的领域的专用名词来描述问题。或者可以尽量使用计算机科学领域的专用名词

其他

  • 尽量不要添加没有用的语境/前缀
  • 单词的描述尽量保证精确性
  • 不要害怕重命名，想到更好的词要立即替换过来

Freedom and discipline

自由与自律


所谓自由，不是随心所欲，而是自我主宰。 －－康德


词典中对自由的定义是：在法律规定的范围内，随自己意志活动的权利。康德的道德哲学理论中，人作为理性存在物，其意志应当是自由的，自由是人的一种天赋权利，是每个人由于他的人性而具有的独一无二的、原生的、与生俱来的权利。一个人只有作为纯粹意志存在物而不是作为一个自然的存在物，他才是自我决定的，才能“为自己立法”，才能在立法时不服从异己意志。

简而言之，人的意志是自由的，人可以不受约束地自我思考，在道德上自我决断，在行动上自我选择，在后果上自我负责。发现自己、把握自己、为自己的选择所带来的后果承担责任。必须强调的是，自由与责任密不可分，自由的连带方面是自我负责，承担起与你的自由选择相关的一切责任。换言之，自由的时代也是个人责任的时代，要自由就必须为自由选择的后果负责－－－无论是成功还是失败。 个人责任没有替代品。

而其实真正想说的是自律。

既然人是自由的，那么人可以按自己的想法做法律范围内的任何事。但是是不是想做什么就做什么这样就好呢？当然不是。

人性有其固有的弱点，当一个人无法用自律控制自己的意志时，他做的事情总是在受习惯和即时诱惑的影响，要么就是被他人的思想所困扰，几乎无法追寻到自己的内心或者永远做不了内心真正渴望的事情。

举个例子，人有两个最折磨人的欲望：一是食欲，二是性欲。

放纵食欲致人肥胖，早晚失去健康。放纵性欲使人或染病、或空虚、或愧疚、或萎靡。

人在做决定的时候总会选择对自己效用大的，权衡了A与B，A获得的效用比B大，那么选择A。然而人又总是短视或是侥幸的，“先吃完这顿大餐再说”， “偶然吃一次胆固醇不会升高”，不知不觉，一顿大餐就吃了标准值的三倍热量，体重控制不住地增长。

私以为人之修养的最高境界为“克己慎独”，人始终要自律。自律是欲望的延时满足。别忘了人之所以高于动物，是因为人能控制自己的思想。往往自律带来的满足比欲望一时的满足要大得多，且衰退期长。计算机领域中，很多最优化算法并不是在每一步都选取了局部最优解，在每个阶段可能只是选取了局部次优或者更次的解，然而从全局的角度来看，最终顺利达到了最优的效果。如果从一开始就在每一步贪心地选择局部最优解，可能早就失去了达到全局最优的机会。

上面说过自由与责任密不可分。暴食、纵欲、出轨、骄奢、懒惰、阴谋等这都是个人的自由，而放纵这些所带给你的快乐享受都不是免费的。肥胖、疾病、身败名裂、穷困潦倒总会在这之后加倍偿还。不自律者，永远只是自己欲望的奴隶。

自由是将选择权交给自己，同时也把对结果的责任交给了自己。没有自律，就没有真正的自由。

pointer parameter passing and allocate memory

Posted on 2016-09-01 Edited on 2019-01-19 In programming

指针做形参传递及进行内存分配


内存的分配方式

  1. 从静态存储区分配。内存在编译期已经分配完成，此块内存存放在data数据段中，在进程的整个运行期都存在，例如静态变量，全局变量。
  2. 从栈区分配。函数的局部自动变量都可以在栈中创建，函数执行结束后这些存储单元将被自动释放。栈空间分配运算内置于处理器的指令集中，效率很高，但是分配的容量有限。
  3. 从堆区分配。也称为动态内存分配，程序可以使用malloc申请任意大小的内存，并负责用free释放内存。

指针参数

编译器会为函数的参数制作临时拷贝,指针参数p的拷贝是_p,编译器使_p = p, 如果函数体的程序修改了_p的内容,就导致参数p的内容作了相应的修改,这就是指针可以用作输出参数的原因。

使用指针参数获取内存分配

在C程序的编写过程中,经常需要写一些函数进行内存的分配,常常会在函数参数中使用指针参数传递内存块的地址

  • 一级指针作形参：首先形参和传递的参数是两个不同的变量,即使同名依然不同。指针传递的是一个变量或者一个值的地址,本身还是采用的值传递的方式。即时不能使他指向另一块地址,但是可以改变它指向空间里的值。
  • 二级指针作形参：二级指针也是传值,但它指向的是一个一维指针,所以函数中可以改变二级指针所指向的一级指针的内容,但是不能改变二级指针本身的内容。所以二级指针传递的是一个一级指针。

核心就是,要改变一个变量的值,就要将这个变量的地址传递给函数,不用管这个变量存储的是值还是指针地址


下面看几个具体的程序

get1.c

void get1(int *p, int num) &#123;
    p = (int *)malloc(num * sizeof(int));
    return;
&#125;

int main() &#123;
    int *p = NULL;
    get1(p, 5);
    if (p == NULL) &#123;
        printf("allocate failed.\n");
    &#125; else &#123;
        printf("allocate success.\n);
    &#125;
&#125;

这段代码输出

allocate failed.

main函数将指针p传给函数get1,并在函数内部创建p的副本_p,而函数中只是改变了_p的地址,而main中的p丝毫未变。且这段程序每执行一次,malloc就会泄漏一段内存,因为使用完后没有释放。

上面我们说过,要改变一个变量的值,就要将这个变量的地址传给函数,get1.c中我们想改变的是指针p,那么我们就应该把指针p的指针pp传递给函数,这样函数在修改指针pp的时候,实质上就修改了指针p的值,下面是修改后的代码get2.c

get2.c

void get2(int **pp, int num) &#123;
    *pp = (int *)malloc(sizeof(int) * num);
    (*pp)[1] = 2;
    return;
&#125;

int main() &#123;
    int *p = NULL;
    get2(&p, 5);
    if (p == NULL) &#123;
        printf("allocate failed.\n");
    &#125; else &#123;
        printf("allocate success.\n);
    &#125;
&#125;

这样就起到了正确的效果,指针p的地址被传递给函数,函数中修改了指针p的值并分配了内存,程序也得到了正确的结果。

allocate success.

由于多级指针的概念比较绕,在写较为复杂的程序时不容易理解且容易出错,可以使用返回值的方式来传递分配好的动态内存,下面是代码示例get3.c

get3.c

int* get3(int *p, int num) &#123;
    p = (int *)malloc(num * sizeof(int));
    p[1] = 2;
    return p;
&#125;

int main() &#123;
    int *p = NULL;
    p = get3(p, 5);
    if (p == NULL) &#123;
        printf("allocate failed.\n");
    &#125; else &#123;
        printf("allocate success.\n);
    &#125;
&#125;

注意,使用这种方法分配内存的时候只可以使用malloc分配动态内存, 因为malloc分配的内存是位于上面所说的堆区,在函数执行结束后仍然存在于堆区中,直到手动用free释放它。而如果用定义自动变量的方式去分配内存将会遇到错误,因为自动变量随函数空间分配在栈区中,一旦函数执行完毕将被销毁,函数内分配的空间也将被收回。

上面用了简短的篇幅和3个简单的例程介绍了指针参数传递内存的基础知识。最后,C程序中内存管理操作决定了程序生命的稳定,珍爱生命,小心内存。
