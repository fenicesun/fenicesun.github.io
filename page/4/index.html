电台情节


当我还是一个很年轻的中学生的时候，手里不时地能拥有一个收音机。

德生，安健，或者是集成在其他设备上的。

最初只是因为年幼家贫手里没有任何可把玩的数码产品，却能用两节5号电池听到外面的声音而感到兴奋。

我喜欢电台里的音乐，尤其是那些深情款款的情歌，并且讨厌AM频道里“滋滋”的杂声。

初中的时候在县城读书，父母给我在县中对面租了间小房子。那时候家里还不是很宽裕，不可能有高级的数码产品。每天早晨父亲都会在给我准备早饭的时候打开收音机的淮安台，那是一档音乐时间，还记得那时的场景。略有凉意的秋天，早晨的雾气很重，6点10分左右收音机里就会传来王菲那首深情的我愿意。

“我愿意为你”

初中没有晚自习，每天晚上吃完饭的时候习惯性的把电台调到FM建湖广播，30分钟的点歌时间，虽然没有什么好的音乐，我还是一直撑到节目的结束，因为我实在没有其他途径能听到流行歌曲了。倒是那个年代电台里总有人点一些过于流行的歌，听了多了有种俗气的感觉。记得最深有首叫快乐崇拜的歌，每天都放听到厌烦。

初中懵懂、孤独而欢乐着的时光不经意间就过去了，至今还记得每逢考试前祈祷的场景。 高中依然很幼稚，那三年喜欢了一个女生，被第一眼的感觉折磨地死去活来直到多年以后，其实还是在自己的世界里想着自己的事而别人没有理会而已。

高一寒假把姑姑给我的剩下的钱买了台安健的收音机，120块，其实那时手上已经有很多数码设备了，有了属于自己的笔记本电脑，mp4,还有学习机什么的。只是更喜欢电台。

那时候的生活还是很规律，尽管叛逆，孤独。只要有时间，每到晚上九点半就会听盐城交通广播台一档叫“都市夜心情”的节目，失意的人们讲述自己的不幸的故事，那个叫恒非的男主播会一一开导他们，教导他们如何对待感情和生活，是那样的理性。有时男主播也会读一段感人的情感故事，我静静聆听。当时只是关注那些人们的故事，企图从他们身上找到自己故事的翻版，偶尔也有罢了。很可惜我高二的时候那档节目已经没有了，也不知道调整到了哪个时间段，只知道男主播去了多伦多高中当了教师。

炜炜动听是我很喜爱的一个节目，这是一档点歌音乐节目，音乐非常不错。坚持听这档节目的更深原因是节目名和我当时疯狂喜欢的一个女生的名字一样。我高二时那个女主播好像已经29岁, 那时是09年。

后来还听过很多不同的电台节目，中学那段迷茫的年代短短续续地听着解着闷。时常想象着要是有一天我也能去主播一个节目，放着我喜欢的音乐，读着我喜欢的文章，讲述我喜欢的故事，这是一种多么不错的感觉。不过逐渐忙碌后也只能从别人的文章里看到我电台的记忆了。

可能在我的梦里，我也是一个电台主播。

那年的夜空没有光


很多年之前，当我还是个小孩子的时候，约莫2年级到5年级的光景，夜晚的天空是没有光的。

那些年还住在郊区，周围都是各式的厂子，上班的工人都是临近的农村的人们，每到晚上8点过后，记忆中除了一家水泥厂，其他的厂房都陷入一片寂静和黑暗之中。

那时候我家就在水泥厂的家属区，居住在那儿的人还很多，但是一到晚上，整个家属区立刻黑的没影。

记忆中那时的星星是灿烂而明亮的，就像挂在天空的璀璨的钻石，纯净而闪耀在我心里。

那些年我妈还在水泥厂上班，夜班时每晚12点起来去工作，日子艰难得很

有时一个人在自己房间里仰望着天空，年幼的时候，没有那么多的事情需要去思考。没有那么多梦，没有那么多追逐，没有爱过谁，也还有很多简单的快乐。我想着的也就是周末怎么打发以及什么时候才能有那一个机器人玩具或者四驱车。

那时的心境就像那时的天空，纯净的一丝不染。

时常下乡去外婆家，乡村的夜晚，记忆格外清晰。

乡村的夜晚，7点以后只有各家窗户点点从电视机里透出的光，偶尔有晚归的人打上手电行走在砖泥小路上。我从不敢在夜晚走在那里的小路，连出堂屋的门都觉得害怕，一是年纪小，二是晚上真是什么都看不到，在那种离城市无限远的地方。

大人说晚上不能出去，会遇到鬼火，我吓着了。

记忆很深的一个场景，一个夜晚，晚上妈妈和外婆在厨房忙着，我站在堂屋前的天井里，呆呆地望着天，一片漆黑，除了那几棵遥远而闪烁的星星，尽情绽放他们的光芒。远处不知道哪里的电线杆上的大喇叭广播，正在放着当时很火的一部叫《萧十一郎》的电视剧的主题曲，听的我入迷。很多年后我知道那首歌叫《转弯》，都是后话了。

那些年的我年幼而无知，没有零用钱也没有城里孩子的各种，沉浸在自己的世界里并自得其乐，不会去想着哪家公司或是上哪个大学，不会因为想到某个人而彻夜难眠。那时的天空尚未开化，没有太多的光。偶尔看着星星都能觉得满足，虽然全家生活的还不是很好，然是快乐着。

随着年岁的慢慢增长，到城里上了学，后来家也搬进了城里，爱过某个人，认识了很多人，上了大学，找了工作。 但是各种压力和忙碌伴随而来，并且享受着他们，天空逐渐亮了起来，光的强度也越来越大，直到发现，似乎夜空就是亮的。

我知道这是成长和生活必需的一步。伤感而无措。

想起那年的夜空没有光。

a guide to when to use reference parameters in cpp

Posted on 2012-05-15 Edited on 2019-01-18 In programming

何时在C++中使用引用参数

引用是变量的别名，与目标变量共同指向目标变量的内存地址，对引用的操作和对变量的操作完全一样。使用引用的主要目的在于函数参数的传递中，解决大块数据或对象的传递效率问题。

引用作为参数

使用引用传递函数的参数，在内存中并没有产生实参的副本，它是直接对实参操作；而使用一般变量传递函数的参数，当发生函数调用时，需要给形参分配存储单元，形参变量是实参变量的副本；如果传递的是对象，还将调用拷贝构造函数。因此，在传递参数的字节数较大时，用引用参数比一般变量传递参数省去了变量拷贝的内存分配和时间开销，效率更高。使用指针作为函数参数和与使用引用参数效果类似，但是调用函数中同样要给形参（指针变量）分配储存单元，且容易产生错误且程序阅读性较差。

指导原则

  1. 对于使用传递的值而不做修改的函数:

    • 如果数据对象很小，如内置数据结构或小型结构，则按值传递。
    • 如果数据对象是数组，则使用指针。
    • 如果数据对象是较大的结构，则使用const指针或者const引用，以提高效率，节省复制结构所需的时间和空间。
    • 如果数据对象是类对象，则使用const引用。传递类对象参数的标准方式是按引用传递。

  2. 对于修改调用函数中数据的函数:

    • 如果数据对象是内置数据类型，则使用指针。
    • 如果数据对象是数组，则只能使用指针。
    • 如果数据对象是结构，则使用引用或指针。
    • 如果数据对象是类对象，则使用引用。
